# JLU Agent Gateway v2.0 使用指南

## 🚀 快速启动

### 1. 启动服务
```bash
node index-v2.js
```

### 2. 访问地址
- **主页**: http://localhost:3000 （自动跳转到前端）
- **现代化前端**: http://localhost:3000/frontend/index.html
- **健康检查**: http://localhost:3000/api/health

---

## 🎯 功能模块说明

### 模块A：极速爬虫 (Scraper Agent)
**功能**: 自动爬取吉林大学OA系统的校园通知

**使用方法**:
1. 在前端界面点击"🚀 极速爬虫"
2. 设置目标意图（如："本科生院通知"）
3. 选择爬取数量（5/10/15/20条）
4. 点击"启动极速爬虫"

**API调用**:
```bash
# 普通模式
curl http://localhost:3000/api/notices

# SSE流式模式（实时进度）
curl http://localhost:3000/api/notices/stream
```

---

### 模块B：智能对话 (Chat Controller)
**功能**: 基于记忆库的智能问答系统

**使用方法**:
1. 在前端界面点击"💬 智能对话"
2. 输入问题（如："最近有什么本科生院的通知？"）
3. 点击"发送"

**API调用**:
```bash
# 普通模式
curl -X POST http://localhost:3000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"query": "最近有什么本科生院的通知？"}'

# SSE流式模式（打字机效果）
curl -X POST http://localhost:3000/api/chat/stream \
  -H "Content-Type: application/json" \
  -d '{"query": "最近有什么本科生院的通知？"}'
```

---

### 模块C：记忆管理 (Vector DB)
**功能**: 查看和管理已爬取的通知数据

**使用方法**:
1. 在前端界面点击"💾 记忆管理"
2. 点击"刷新列表"查看所有记忆
3. 可以查看记忆统计、清空记忆等

**API调用**:
```bash
# 获取记忆统计
curl http://localhost:3000/api/stats

# 获取所有记忆
curl http://localhost:3000/api/memories

# 清空记忆
curl -X POST http://localhost:3000/api/memories/clear
```

---

### 模块D：系统统计 (Stats)
**功能**: 查看系统运行状态和架构信息

**使用方法**:
1. 在前端界面点击"📊 系统统计"
2. 查看5大模块的运行状态

**API调用**:
```bash
curl http://localhost:3000/api/health
```

## 🔧 常见问题

### Q1: 前端页面无法访问？
**A**: 确保使用正确的URL：
- http://localhost:3000 （自动跳转）
- http://localhost:3000/frontend/index.html

### Q2: 爬虫速度慢？
**A**: v2.0已优化：
- 资源拦截（屏蔽图片/CSS/字体）
- 并发处理（5批次并发）
- 预计速度提升5-10倍

### Q3: 返回乱码？
**A**: 这是Windows终端编码问题，不影响功能。请使用：
- 现代化前端界面（推荐）
- 或在浏览器中直接访问API

### Q4: 如何更新监控意图？
**A**: 两种方式：
1. 前端界面输入新意图
2. API调用：
```bash
curl -X POST http://localhost:3000/api/intent \
  -H "Content-Type: application/json" \
  -d '{"intent": "本科生院通知"}'
```

## 🏗️ 架构说明

### 5大核心模块
- **模块A: 探路者 (Scraper Agent)** - 极速爬虫引擎
- **模块B: 潜水员 (Deep-Dive Agent)** - 并发深度爬取
- **模块C: 文件粉碎机 (Document Parser)** - 多格式文件解析
- **模块D: 记忆中枢 (Vector DB)** - 向量数据库存储
- **模块E: 前台接待员 (Chat Controller)** - 智能对话系统

### 技术栈
- **后端**: Node.js + Express
- **爬虫**: Playwright（资源拦截优化）
- **存储**: JSON向量数据库
- **AI**: OpenRouter (DeepSeek/StepFun)
- **前端**: TailwindCSS现代化界面
- **通信**: REST API + SSE流式传输
